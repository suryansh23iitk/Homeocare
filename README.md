# Homeocare
An app developed locally for arranging patient history and prescriptions
Build a clean, secure, and user-friendly patient record management mobile app for a homeopathy clinic.

The app should allow doctors to:

1. Add new patients with:

- Unique patient ID/number (auto-generated)
- Name
- Age
- Gender
- Phone number
- Address
- Notes/history

2. Upload and store patient documents in the form of:

- Photos/images
- PDFs
- Lab reports
- Prescriptions
- Scans/X-rays

3. Create a searchable database with a fast search bar that can search patients by:

- Patient ID
- Name
- Phone number

4. Open each patient profile to view:

- Uploaded files
- Visit history
- Follow-up notes
- Date-wise timeline

5. Include a prescription designer and printable prescription feature:

- Create digital prescriptions
- Add clinic logo, doctor name, qualification, registration number, address, and contact details
- Add diagnosis, symptoms, medicines, dosage, advice, and follow-up date
- Save prescription templates
- Auto-fill patient details into prescription
- Generate clean printable PDF prescriptions
- Support A4 prescription format
- Enable direct printing and WhatsApp sharing
- Include signature area and clinic branding
- Option for both modern and classic prescription layouts

6. Include:

- Simple modern UI
- Secure login/password protection
- Cloud backup/storage
- Fast loading
- Mobile-friendly design
- Easy file upload from camera, gallery, or files

Preferred tech stack:

- Flutter frontend
- Firebase backend
- Firestore database
- Firebase Storage for PDFs/photos

Additional optional features:

- OCR text extraction from reports
- Appointment reminders
- WhatsApp integration
- Offline access
- Multi-device sync
- Analytics/dashboard
- Medicine history tracking
- Follow-up reminders

The UI should feel minimal, elegant, professional, and easy for doctors to use during busy clinic hours. The app should reduce paperwork while making patient record retrieval and prescription management extremely fast and organized. The current bug is that each patient is getting assigned same unique ID.
